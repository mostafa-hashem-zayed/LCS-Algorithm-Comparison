How Each Algorithm Works
Recursive + Memoization
Breaks the problem into subproblems top-down. When characters match, recurse diagonally; otherwise take the best of two branches. Results cached to avoid recomputation.
Time: O(m·n)   Space: O(m·n)
Brute Force
Generates all 2ⁿ subsequences of string A, then checks each one against string B. Correct but devastatingly slow — impractical beyond ~20 characters.
Time: O(2ⁿ · m)   Space: O(2ⁿ)
Dynamic Programming
Fills an (m+1)×(n+1) table bottom-up. Each cell stores the LCS length of the prefixes seen so far. Backtrack from dp[m][n] to reconstruct the actual subsequence.
Time: O(m·n)   Space: O(m·n)
